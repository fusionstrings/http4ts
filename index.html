http4ts

Server as a Function http toolkit for TypeScript & JavaScript

Get Started

Server as a Function

Model your server application as a single function; Based on the Twitter paper Your Server as a Function and inspired by the fantastic http4k library.

Runtime Independence

The core library does not have any dependency on the Node.js environment. It should be possible to use it for both Node.js and Deno. Node.js bindings are available out of the box.

Symmetric

Symmetric interfaces for the HTTP client and HTTP server. It is possible to reuse the same HttpHandler interface and all the filters on both server- and client-side. There is an HttpClient functionality available in the library which follows the fetch interface and is independent of any runtime.

Type Safety

built using the maximum type safety power of TypeScript

Immutability

all entities in the library are immutable unless, naturally, it is not possible.

Testability

The basic building blocks of this library are functions and the main entities are abstracted from the environment, it is extremely simple to write tests for the code built by http4ts.
